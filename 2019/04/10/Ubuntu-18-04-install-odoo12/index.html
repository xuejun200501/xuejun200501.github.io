Ubuntu 18.04 下安装 Odoo 12

Posted on 2019-04-10 | In technology

前言

根据公司要求搭建odoo开源企业管理系统

软件需求

  • odoo12
  • ubuntu 18.04

安装

安装前准备

  1. 更新系统最新安装包

  2. 安装pip, git, node.js 和其他odoo必须依赖工具

创建odoo 用户

  1. 创建系统用户odoo12(根据自己需求创建)

安装 PostgreSQL

  1. 使用Ubuntu默认repo安装

  2. 创建PostgreSQL 用户，和系统用户相同的名字

安装Wkhtmltopdf

wkhtmltopdf 能够把多个“对象”合并生成一个pdf文档，这些“对象”可以是“页面对象”、“封面对象”、或是“目录对象”。这些对象在pdf文档中的顺序可以通过命令行参数来指定。命令行参数包括两部分，一种是针对某一特定“对象”的命令行参数，另一种是全局的命令行参数。并且全局的命令行参数只能放在全局参数区(GLOBAL OPTION)中指定。
官方wkhtmltopdf

  1. 使用wget下载

  2. 安装

安装odoo及进行配置

  1. 切换用户

  2. 去GitHub下载最新的代码

  3. 创建python虚拟环境

  4. 激活虚拟环境

  5. 下载必要的模块

  6. 退出虚拟环境

  7. 创建一个用于存放自定义odoo模块的文件夹

  8. 创建配置文件

创建一个启动文件（Systemd Unit File）

注：修改Sytemd后必须重启
odoo12@odoo:/opt/odoo12$ sudo systemctl daemon-reload

开启odoo服务

注：
设置开机启动
odoo12@odoo:/opt/odoo12$ sudo systemctl enable odoo12
查看日志
odoo12@odoo:/opt/odoo12$ sudo journalctl -u odoo12

遇到问题：

  1. 安装好后，无法打开

    解决：查看是不是防火墙8069端口没有添加进ufw
    odoo12@odoo:/opt/odoo12$ sudo ufw status

  2. 500错误

    解决：查看是否是Postgres没有启动

  3. odoo备份数据出错，提示Postgresql 9.2 pg_\dump version mismatch（CentOS）

    解决：这种问题，主要是系统自带有PostgreSQL，此版本比较低，而我们又安装了新的PostgreSQL所致，可以使用命令查看下:

    可以看到pg_dump有二个地方存在 /usr/bin/和/usr/pgsql-10/bin/
    去这二个目录下查看下(pg_dump -V)，如果二个版本不同，卸载系统自带的
